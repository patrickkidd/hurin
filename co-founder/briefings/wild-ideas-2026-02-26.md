**TECHNOLOGY WILDCARDS**

- **Agentic Clinical Interview — The AI Drives the Genogram.** Right now chat.py calls `gemini_text_sync()` with a static conversation flow prompt. But your prompt already describes 7 phases with turn counts. Make this dynamic. After each user response, the AI evaluates: "What's missing from the genogram? What phase am I in? What's the highest-value question right now?" It checks the current PDP state (people extracted, events coded, gaps in the family map) and chooses the *most informative* next question. This is an *agent*, not a chatbot. The difference: a chatbot follows a script. An agent has a *goal* (complete family map with SARF variables coded) and *reasons* about how to achieve it. Implementation: add a `_choose_next_question()` function that takes current PDP + conversation history and returns a question strategy. Feed that strategy as context to the response generation call. The coverage map idea above gives you the objective function — minimize uncovered family nodes.

- **Offline-First with On-Device Extraction.** The Personal app already stores data locally as JSON (not SQLite, not ORM — just dicts). The chat and extraction currently require network calls to btcopilot's Flask server. But Gemini Nano runs on-device on Android, and Apple Intelligence runs locally on M-series. What if the extraction pipeline ran entirely on the user's phone? Your `_extract_and_validate()` function (pdp.py:527-576) is 50 lines of validation logic + a Gemini call. Swap `gemini_structured()` for a local model call and you have HIPAA-compliant extraction with zero PHI transmission. The audio-upload-flow.md doc identifies 7 HIPAA compliance risks — **every single one** is solved by going on-device. This is the nuclear option for the healthcare market.

- **Scene.py → Graph Database.** Your 3,012-line god object (scene.py) is fundamentally a graph — People are nodes, PairBonds/ChildOf/Emotions are edges, Events are annotated hyperedges. You're maintaining a custom graph in Python dicts with manual cascade deletes, undo tracking, and cross-reference management.
